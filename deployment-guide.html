Feedback Widget — Deployment Guide

Four steps to get the feedback widget live and attached to the AI Paraphrasing prototype. Total time: ~20 minutes.

1
Set up Supabase
~5 min

Supabase stores the feedback pins. It's free, no credit card required.

  1. Go to supabase.com → sign up → click New project
  2. Name it build-au-feedback, choose a region close to Australia
  3. In the sidebar go to SQL Editor and run this query:
create table feedback_pins ( id uuid default gen_random_uuid() primary key, prototype_id text not null, x_pct float not null, y_pct float not null, comment text not null, author_name text, element_context text, screenshot text, created_at timestamptz default now() ); alter table feedback_pins enable row level security; create policy "Anyone can insert" on feedback_pins for insert with check (true); create policy "Anyone can read" on feedback_pins for select using (true); create policy "Anyone can delete" on feedback_pins for delete using (true);
  1. Go to Project Settings → API
  2. Copy the Project URL and the anon / public key — you'll need both in Step 2
2
Configure the widget
~2 min

Open FeedbackWidget/feedback-widget.js and fill in your Supabase credentials at the top of the file:

// ═══ CONFIGURATION — fill these in ═══ const SUPABASE_URL = 'https://xxxxxxxxxxxx.supabase.co'; const SUPABASE_ANON_KEY = 'eyJhbGci...'; const ADMIN_PASSWORD = 'B3tterSoftware2026!'; // change if you like
Note on the anon key The anon key is designed to be public — it's safe to commit it to GitHub. Supabase's Row Level Security policies control what it can actually do.
3
Create GitHub repo and deploy to Vercel
~10 min

The FeedbackWidget folder is ready to be pushed as a GitHub repo. It contains:

deployment-guide.htmlThis file

On GitHub

  1. Go to github.com → New repository
  2. Name it build-au-feedback-widget (or similar)
  3. Make it Public (Vercel free tier requires public repos, or use your Vercel team account for private)
  4. Push the contents of the FeedbackWidget folder as the repo root

On Vercel

  1. Go to vercel.com → Add New Project
  2. Import the build-au-feedback-widget GitHub repo
  3. No build settings needed — Vercel detects it as a static site
  4. Click Deploy
  5. Note your deployment URL — by default it will be something like build-au-feedback-widget.vercel.app
You can set a custom domain in Vercel if you want a cleaner URL, e.g. feedback.yourdomain.com.
4
Connect to the Paraphrasing prototype
~2 min

The script tag has already been added to Paraphrase/deploy/index.html. It currently points to:

<!-- near the bottom of the file, before </body> --> <script src="https://build-au-feedback-widget.vercel.app/feedback-widget.js"></script>

If your Vercel URL is different from build-au-feedback-widget.vercel.app, update the URL in that script tag. Then redeploy the paraphrasing prototype to Vercel as normal.

To add to future prototypes

Add this single line before </body> in any prototype HTML file — that's it:

<script src="https://build-au-feedback-widget.vercel.app/feedback-widget.js"></script>
✓
Once live — how it works
  • Reviewers open the prototype URL, click Feedback, click anywhere to drop a pin, enter their name and comment → saved to Supabase instantly
  • You click Review, enter password (B3tterSoftware2026!), see all pins with screenshots and element context
  • Click Export for Claude to copy a formatted summary you can paste directly into Claude for AI-assisted analysis
  • Click ✓ Resolved — delete to clear addressed items
  • Pins are scoped by URL path — the same widget file serves multiple prototypes without comments mixing
